Hãy tạo cho tôi một ứng dụng web chat realtime gồm backend Spring Boot.

Yêu cầu tổng quan:
- Ứng dụng chat realtime cho khoảng 6 - 8 người dùng.
- Không sử dụng database quan hệ như MySQL/PostgreSQL.
- Chỉ sử dụng Redis để lưu trạng thái và lịch sử chat.
- Backend dùng Spring Boot.
- Frontend dùng ReactJS.
- Giao tiếp realtime bằng WebSocket/STOMP.
- Cung cấp sẵn 6 - 8 userId mẫu để người dùng chọn khi vào web chat.
- Người dùng chọn userId, nhập roomId, sau đó khởi tạo socket để vào room chat.
- Mỗi room chat lưu tối đa 5000 dòng tin nhắn trong Redis.
- Khi vượt quá 5000 tin nhắn, Redis tự xóa tin nhắn cũ nhất.
- Khi người dùng vào room, frontend load lịch sử chat từ Redis.
- Không cần đăng nhập thật, chỉ cần chọn userId mẫu.

Backend yêu cầu:
- Spring Boot 4.x
- Java 21
- Sử dụng RedisTemplate hoặc StringRedisTemplate.
- Sử dụng WebSocket STOMP.
- Có endpoint REST:
  - GET /api/users: trả về danh sách userId mẫu.
  - GET /api/rooms/{roomId}/messages: lấy lịch sử chat của room.
- Có WebSocket endpoint:
  - /ws
- Client subscribe:
  - /topic/rooms/{roomId}
- Client gửi message tới:
  - /app/rooms/{roomId}/send
- Message object gồm:
  - id
  - roomId
  - userId
  - content
  - createdAt
- Khi nhận message:
  - validate userId không rỗng
  - validate content không rỗng
  - lưu message vào Redis list theo key: chat:room:{roomId}:messages
  - dùng LPUSH hoặc RPUSH hợp lý
  - trim list để chỉ giữ tối đa 5000 tin nhắn mới nhất
  - broadcast message cho các client trong room
- Cấu hình CORS cho frontend ReactJS.
- Cung cấp đầy đủ code các file cần thiết:
  - pom.xml
  - application.yml
  - WebSocketConfig
  - RedisConfig
  - ChatMessage DTO
  - ChatController REST
  - ChatWebSocketController
  - ChatService
  - UserController

Redis yêu cầu:
- Lưu message dạng JSON string.
- Mỗi room có key riêng:
  - chat:room:{roomId}:messages
- Giữ tối đa 5000 message mới nhất.
- Tin nhắn cũ nhất bị xóa khi vượt ngưỡng.
- Có thể dùng Redis List và LTRIM.


Yêu cầu output:
- Trình bày cấu trúc thư mục backend và frontend.
- Viết đầy đủ code từng file.
- Giải thích ngắn cách chạy:
  - chạy Redis bằng Docker
  - chạy backend Spring Boot
  - chạy frontend ReactJS
- Cung cấp lệnh Docker để chạy Redis:
  docker run --name redis-chat -p 6379:6379 -d redis:latest
- Không bỏ sót file quan trọng.
